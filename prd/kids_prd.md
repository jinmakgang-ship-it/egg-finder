# 키즈알림 (KidsAlert) — 어린이집 알레르기 사전 알림 서비스 PRD

> **버전**: v1.0
> **작성일**: 2026-03-22
> **상태**: Draft

---

## 1. 개요 (Executive Summary)

### 서비스 한 줄 정의
어린이집 식단표에서 아이의 알레르기 유발 메뉴를 자동 감지하여, 제공 **전날 저녁** 부모에게 카카오톡으로 알리고 대체 간식 준비를 안내하는 서비스.

### 해결하는 문제
- 어린이집 식단표는 키즈노트에 올라오지만, 부모가 매일 일일이 확인하지 않으면 알레르기 유발 메뉴를 사전에 인지하기 어렵다.
- 계란 등 특정 알레르기가 있는 아이의 부모는 어린이집에서 제공하는 간식/식사를 아이가 먹지 못할 경우 미리 대체 간식을 준비해야 하는데, 당일이 되어서야 인지하는 경우가 많다.
- 어린이집 교사가 매번 개별 알림을 주기에는 업무 부담이 크다.

### 목표
- 알레르기 아동 부모의 사전 인지율 **90% 이상** 달성
- 알림 발송 후 대체 간식 준비 행동 유도
- 어린이집 교사의 개별 알림 업무 부담 제거

---

## 2. 배경 및 시장 분석

### 시장 현황
- **공백 시장**: 국내 학교 급식 알림 앱(김급식, 모두급식, 배고프당 등)은 NEIS 공개 API 기반으로 초등학교 이상만 지원. **어린이집 대상 알레르기 알림 서비스는 존재하지 않음.**
- 키즈노트는 전국 어린이집의 주요 소통 플랫폼이나 공식 외부 API 미제공.
- 식품 알레르기 시장은 연 9~10% 성장 중 (2035년 글로벌 1,249억 달러 전망).

### 법적 근거
- 어린이집은 보육사업안내에 따라 알레르기 유발물질이 포함된 식단표를 사전에 보호자에게 통지할 의무가 있음.
- 한국 식약처 기준 **19가지 알레르기 유발물질** 번호 표기 의무 (계란 = **1번**).

### 타깃 사용자

| 사용자 | 페르소나 |
|-------|---------|
| **주 사용자** | 계란, 우유 등 알레르기가 있는 영유아(0~7세)의 부모 |
| **부 사용자** | 알레르기 아동을 담당하는 어린이집 교사/원장 |

## 3. 제품 범위 (Scope)

### In-Scope (v1.0)
- 부모가 아이의 알레르기 프로필 등록 (번호 기반, 최대 19개 항목)
- 키즈노트 계정 연결 또는 식단표 수동 입력
- 알레르기 유발 메뉴 자동 감지 (알레르기 번호 파싱)
- 전날 저녁 카카오톡 알림톡 발송
- 알림 메시지에 해당 메뉴명 및 대체 간식 준비 안내 포함

### Out-of-Scope (v1.0 이후)
- 대체 간식 레시피 추천 기능
- 어린이집 B2B 직접 연동
- 영양 분석 기능
- 알레르기 전문 의료 연계
- 다국어 지원

## 4. 핵심 기능 명세

### 4.1 아이 알레르기 프로필 등록

**기능 설명**
- 부모가 앱 가입 후 아이 정보(이름, 생년월일) 및 알레르기 항목을 등록
- 식약처 19가지 알레르기 유발물질 번호 기반 선택 UI 제공
- 복수 알레르기 등록 가능, 이후 수정 가능

**입력 항목**
- 아이 이름
- 어린이집 이름 (검색 또는 직접 입력)
- 알레르기 항목 선택 (체크박스, 1~19번)
- 알림 수신 시간 설정 (기본값: 오후 8시)
- 알림 채널 선택 (카카오톡 / 앱 푸시 / SMS)

**알레르기 번호 선택 UI 예시**
```
☑ 1. 알류 (계란)
☐ 2. 우유
☐ 3. 메밀
☐ 4. 땅콩
☐ 5. 밀
...
```

---

### 4.2 식단표 연동

#### 방식 A: 키즈노트 계정 연결 (자동, 권장)
- 부모가 키즈노트 아이디/비밀번호 입력 (암호화 저장)
- 서비스 백엔드가 매일 오전 7시 키즈노트에서 식단표 자동 수집
- 수집 실패 시 부모에게 수동 입력 요청 알림 발송

#### 방식 B: 식단표 수동 입력 (MVP 단계)
- 부모가 식단표 사진 촬영 업로드 → OCR로 텍스트 추출
- 또는 직접 텍스트 복사·붙여넣기
- 서비스가 알레르기 번호 파싱

**식단표 파싱 로직**
```
입력 예시:
  오전간식: 계란샌드위치 1.5, 우유 2
  점심: 잡곡밥, 된장국 5.8, 계란찜 1, 시금치나물

파싱 결과:
  1번(알류) 포함 메뉴: [계란샌드위치, 계란찜]
  → 알레르기 프로필에 1번 등록된 아이 → 알림 대상
```

---

### 4.3 알레르기 감지 및 알림 발송

**동작 흐름**
```
1. 매일 오전: 식단표 수집 (키즈노트 크롤링 또는 수동 입력)
2. 알레르기 번호 파싱 → 메뉴별 알레르기 목록 생성
3. 각 아동 알레르기 프로필과 비교
4. 해당 알레르기 메뉴가 내일 식단에 있을 경우
5. 당일 저녁 설정 시간에 부모에게 알림 발송
```

**알림 메시지 템플릿 (카카오톡 알림톡)**
```
[키즈알림] ○○이 알레르기 사전 알림

안녕하세요, ○○ 부모님!
내일(3월 23일, 월요일) ○○어린이집 식단에
아이의 알레르기 유발 메뉴가 포함되어 있어요.

⚠️ 알레르기 포함 메뉴
- 오전간식: 계란샌드위치 (알류 1번)
- 점심: 계란찜 (알류 1번)

대체 간식을 내일 등원 시 챙겨주시면 감사하겠습니다.
문의: 담임 선생님께 연락해 주세요.

[키즈알림 앱에서 확인하기]
```

**알림 채널 우선순위**
1. 카카오톡 알림톡 (1순위 — 개봉률 높음, 비용 저렴)
2. 앱 푸시 알림 FCM (2순위)
3. SMS (폴백 — 알림톡 실패 시 자동 전환)

---

### 4.4 알림 이력 및 관리

- 발송된 알림 이력 확인 (날짜별)
- 알림 읽음/미읽음 상태 표시
- 알림 OFF/ON 토글 (임시 중단)
- 알레르기 프로필 수정 (성장하면서 알레르기 변화 반영)

## 5. 사용자 흐름 (User Flow)

```
[신규 가입]
앱 다운로드
→ 회원가입 (전화번호 인증)
→ 아이 프로필 등록 (이름, 어린이집)
→ 알레르기 항목 선택
→ 키즈노트 계정 연결 OR 식단표 수동 입력 선택
→ 알림 시간/채널 설정
→ 설정 완료

[일상적 사용]
매일 저녁 (알레르기 해당 시)
→ 카카오톡/푸시 알림 수신
→ 알림 확인 (내일 메뉴 및 알레르기 정보)
→ 대체 간식 준비
→ 다음 날 등원 시 간식 지참
```

## 6. 기술 아키텍처

### 전체 구조

```
[키즈노트 서버]
      │
      │ Playwright 크롤링 (매일 오전 7시)
      ▼
[식단표 수집 서비스]
      │
      │ 알레르기 번호 파싱 (정규식)
      ▼
[알레르기 매칭 엔진]
  - 아동 프로필 DB 조회
  - 알림 대상자 선별
      │
      ▼
[알림 스케줄러 (APScheduler)]
  - 매일 저녁 설정 시간 실행
      │
      ├──→ 카카오톡 알림톡 API (알리고)
      ├──→ FCM 푸시 알림
      └──→ SMS (폴백)
```

### 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 백엔드 | Python FastAPI | REST API |
| 크롤링 | Playwright + BeautifulSoup | 키즈노트 자동화 |
| 스케줄러 | APScheduler | 경량형, 초기 적합 |
| 알림 (카카오) | 알리고 알림톡 API | 건당 약 8원 |
| 알림 (푸시) | Firebase FCM | 월 6M건 무료 |
| 알림 (SMS) | CoolSMS / 알리고 | 폴백용 |
| 데이터베이스 | PostgreSQL | 사용자/알레르기 프로필 |
| 캐시/큐 | Redis | 작업 큐 |
| 인프라 | AWS Lightsail | 초기 $5~20/월 |
| 앱 | React Native | iOS/Android 크로스플랫폼 |

### 데이터 모델 (핵심)

```
User (부모)
  - id, name, phone, kakao_token

Child (아동)
  - id, user_id, name, daycare_name
  - allergen_list: [1, 2, 5] (알레르기 번호 배열)

MealPlan (식단표)
  - id, child_id, date
  - meals: [{time: "오전간식", menu: "계란샌드위치", allergens: [1, 5]}, ...]

NotificationLog (알림 이력)
  - id, child_id, sent_at, channel, message, status
```

## 8. MVP 범위 및 출시 계획

### Phase 1 — MVP (1~2개월)
**목표**: 핵심 가치 검증

- [ ] 아이 알레르기 프로필 등록
- [ ] 식단표 수동 입력 (텍스트 붙여넣기)
- [ ] 알레르기 번호 파싱 로직
- [ ] 카카오톡 알림톡 발송
- [ ] 알림 이력 확인

**예상 비용**: $20~50/월

### Phase 2 — 자동화 (3~4개월)
**목표**: 자동화로 사용 편의성 향상

- [ ] 키즈노트 계정 연결 및 자동 식단표 수집
- [ ] OCR 식단표 이미지 파싱
- [ ] 알림 시간/채널 커스터마이징
- [ ] 앱 푸시 알림 추가

**예상 비용**: $50~100/월

### Phase 3 — 확장 (6개월+)
**목표**: B2B 전환 및 기능 확대

- [ ] 어린이집 직접 B2B 연동
- [ ] 대체 간식 레시피 추천
- [ ] 다중 아동 관리 (형제 지원)
- [ ] 어린이집 교사용 대시보드

**예상 비용**: $200~500/월

## 10. 리스크 및 대응 방안

| 리스크 | 영향도 | 대응 방안 |
|-------|-------|---------|
| 키즈노트 크롤링 차단 | 높음 | MVP는 수동 입력 제공, 키즈노트 공식 파트너십 협의 병행 |
| 아동 개인정보 보호 이슈 | 높음 | 알레르기 데이터 암호화 저장, 개인정보 처리방침 명시, PIPA 준수 |
| 식단표 알레르기 번호 누락/오기재 | 중간 | 어린이집 별 파싱 오류 모니터링, 수동 수정 UI 제공 |
| 키즈노트 자체 알레르기 알림 기능 출시 | 중간 | 대체 간식 안내, 개인 알레르기 프로필 등 차별 기능 강화 |
| 카카오 알림톡 심사 거부 | 낮음 | 정보성 메시지 형식 준수, 사전 심사 통과용 템플릿 준비 |
